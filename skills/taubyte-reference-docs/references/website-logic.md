# Website Logic

## Website Repositories

- Websites are stored in git repositories
- Repository naming: `tb_website_<name>`
- Can be public or private
- Support multiple branches

## Domain and Path Configuration

- Domains: Domain names that serve the website
- Paths: URL paths that route to the website
- Multiple domains and paths can be configured
- Used for routing HTTP requests

## Repository Operations

Websites support standard git operations:
- Clone: Download website repository
- Push: Upload local changes
- Pull: Download remote changes
- Checkout: Switch branches
- Import: Import existing repository
