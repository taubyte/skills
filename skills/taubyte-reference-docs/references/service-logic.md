# Service Logic

## Service Resources vs Dream Services

- Service Resources: Configuration resources within projects/applications
- Dream Services: Runtime services injected into Dream universes (auth, patrick, etc.)
- These are different concepts with different purposes

## Service Protocol

- Defines the communication protocol for the service
- Examples: http, p2p, etc.
- Configured during creation or editing

## Service Configuration

- Service resources define service behavior
- Part of application or project configuration
- Used by runtime to configure service execution
