# Library Logic

## Library Repositories

- Libraries are stored in git repositories
- Repository naming: `tb_library_<name>`
- Can be public or private
- Support multiple branches

## Library Usage

- Libraries can be used as source for functions
- Referenced using: `library/<library-name>`
- Shared code across resources
- Version controlled

## Repository Operations

Libraries support standard git operations:
- Clone: Download library repository
- Push: Upload local changes
- Pull: Download remote changes
- Checkout: Switch branches
- Import: Import existing repository
